# 社内備品管理システム（OJT ベースプロジェクト）

## アプリの概要

社内の備品（PC・AV 機器など）を管理するデスクトップアプリです。  
備品の一覧確認・貸出・返却・履歴管理などの機能を持ちます。

このプロジェクトは **新人研修用のベースコード** です。  
完成形ではなく、 2 か月かけて機能を追加していくための土台として用意されています。

---

## 動作環境

| 項目 | 内容 |
|------|------|
| フレームワーク | .NET 8 / WPF |
| データベース | SQLite（`Microsoft.Data.Sqlite`） |
| 対応 OS | Windows 11 |
| 開発環境 | Visual Studio 2022 以上 |

## 起動方法

1. `EquipmentManager.sln` を Visual Studio で開く
2. NuGet パッケージの復元を確認する（自動で行われる）
3. `F5` キーまたは「開始」ボタンで実行する

---

## DB ファイルの場所

アプリを初回起動すると、実行ファイルと同じフォルダに DB ファイルが作成されます。

```
（実行フォルダ）/equipment_manager.db
```

Visual Studio でデバッグ実行した場合の例：
```
EquipmentManager\bin\Debug\net8.0-windows\equipment_manager.db
```

---

## 初回起動時の動作

DB ファイルが存在しない場合、自動的に以下の処理が行われます。

1. `equipment_manager.db` の作成
2. テーブルの作成（`Employees` / `Equipment` / `RentalHistories`）
3. 動作確認用サンプルデータの投入

2 回目以降の起動では、既存の DB ファイルをそのまま使用します。  
DB をリセットしたい場合は `equipment_manager.db` を削除してからアプリを起動してください。

## フォルダ構成

```
EquipmentManager/
├── Models/              # データモデル（テーブルに対応するクラス）
│   ├── Employee.cs
│   ├── Equipment.cs
│   └── RentalHistory.cs
├── Data/                # DB 接続・初期化
│   ├── DatabaseConfig.cs       # 接続文字列の一元管理
│   └── DatabaseInitializer.cs  # テーブル作成・サンプルデータ投入
├── Services/            # DB アクセスロジック
│   └── EquipmentService.cs
├── Views/               # 画面（XAML + コードビハインド）
│   ├── TopView.xaml / .cs          # トップ画面
│   ├── MenuView.xaml / .cs         # メニュー画面
│   └── EquipmentListView.xaml / .cs # 備品一覧画面
├── App.xaml / .cs       # アプリのエントリポイント
└── EquipmentManager.csproj
```

## 実装の参考にしてほしいポイント

- **DB 接続文字列** は `Data/DatabaseConfig.cs` の 1 か所で管理しています。
- **DB アクセスロジック** は `Services/` フォルダのクラスに書いてください。画面（View）に SQL を直接書かないようにしましょう。
- **SQL パラメータ** は必ず `@変数名` 形式を使ってください（SQL インジェクション対策）。  
  `EquipmentService.GetAll()` を参考に実装してください。
- **例外処理** は `try-catch` で囲み、`MessageBox.Show` でエラー内容を表示してください。
- **NULL 値** は `reader.IsDBNull(列番号)` で確認してから読み取ってください。
